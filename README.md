# etcd-leyu-pages

本仓库用于归档和发布多个独立的 HTML 页面。这些页面内容不特定于某个域名或网站，而是作为通用 HTML 资源的存储与展示空间。

## 项目简介

`etcd-leyu-pages` 是一个静态 HTML 页面集散仓库。你可以在这里找到各类独立页面，它们可能用于演示、参考或特定场景下的展示。仓库的主要目的是提供一种简单、结构化的方式来托管和分享这些页面。

## 目录说明

```
.
├── index.html          # 仓库入口页面（如有）
├── pages/              # 页面存放目录
│   ├── example1/       # 示例页面1
│   │   └── index.html
│   ├── example2/       # 示例页面2
│   │   └── index.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
├── README.md           # 本文件
└── ...
```

- **`pages/`**：存放所有独立 HTML 页面的目录，每个子文件夹代表一个独立页面。
- **`assets/`**：存放页面可能共享的静态资源文件。

## 页面归档说明

每个独立页面应包含其完整的内容与样式，以便在无外部依赖的情况下正常运行。页面可以引用 `assets/` 目录下的公共资源，但建议尽量自包含，以确保归档的稳定性。

页面命名应简洁明了，避免使用特殊字符或空格。如有多个版本，建议在文件夹内使用 `v1`、`v2` 等子目录进行区分。

## 维护说明

- 新增页面：在 `pages/` 下创建新文件夹，放入 `index.html` 及相关文件。
- 更新页面：直接修改对应文件夹下的文件，注意保持向后兼容。
- 删除页面：移除对应文件夹，并检查是否需同步更新其他引用。
- 提交规范：建议使用有意义的提交信息，如“添加/更新/删除某页面”。

## 其他

本仓库仅为静态页面归档用途，不涉及动态服务或后端逻辑。所有页面内容由贡献者自行维护，仓库管理员负责目录结构和基本规范的执行。
